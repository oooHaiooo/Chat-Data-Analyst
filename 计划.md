#### 📂 Day 2：数据上传与基础对话（数据接入日）
**目标：** 用户能上传 Excel/CSV，且 AI 能“看懂”这个文件包含什么数据。

1.  **文件上传组件（上午）：**
    *   在 Streamlit 中使用 `st.file_uploader`。
    *   用 Pandas 读取上传的文件 (`pd.read_csv` 或 `read_excel`)。
    *   在页面展示前 5 行数据 (`st.dataframe`)。

2.  **构建 Smart Dataframe（下午）：**
    *   引入 `PandasAI`。
    *   配置 LLM（把 Day 1 的 API Key 放进去）。
    *   **关键代码逻辑：**
        ```python
        from pandasai import SmartDataframe
        from pandasai.llm import OpenAI # 或者 DeepSeek 的配置

        llm = OpenAI(api_token="你的KEY") # 如果用DeepSeek需配置base_url
        df = pd.read_csv(uploaded_file)
        sdf = SmartDataframe(df, config={"llm": llm})
        ```

3.  **测试理解能力（晚上）：**
    *   输入问题：“这个表里有哪些列？主要讲了什么？”
    *   验证 `sdf.chat("问题")` 是否能返回正确的文本描述。

**Day 2 产出：** 一个能上传文件并预览表格的网页，后台已经连接了 AI 分析引擎。

#### 📊 Day 3：核心功能 —— 对话生成图表（攻坚日）
**目标：** 实现 Text-to-Plot（文字转图表），这是项目最酷炫的地方。

1.  **聊天输入框（上午）：**
    *   使用 Streamlit 的 `st.chat_input` 和 `st.chat_message` 组件，做一个像微信一样的聊天界面。

2.  **打通绘图链路（下午）：**
    *   用户输入：“帮我画出销售额随时间变化的折线图”。
    *   后端调用 `sdf.chat(user_input)`。
    *   **PandasAI 的魔法：** 它会自动生成 Python 代码 -> 执行代码 -> 生成图片。
    *   你需要处理图片的显示。PandasAI 通常会把图片保存在临时目录，你需要让 Streamlit 读取并显示这张图。

3.  **调试 Prompt（晚上）：**
    *   找几个数据集测试（比如电商数据、成绩单）。
    *   如果 AI 画图报错，尝试在 Prompt 里加提示（System Prompt），比如：“请优先使用 Matplotlib，中文请使用 SimHei 字体防止乱码”。

**Day 3 产出：** 你输入“画个图”，屏幕上真的出现了一个图！

### 💡 几个“作弊”提效技巧（AI First 的精髓）

1.  **遇到报错直接问 AI：**
    不要自己去 StackOverflow 搜，直接把报错信息复制给 ChatGPT/DeepSeek：“Streamlit 报这个错怎么修？”
2.  **让 AI 写前端：**
    不要自己手敲 Streamlit 的布局代码。告诉 AI：“我要一个 Streamlit 页面，左边是侧边栏上传文件，右边是聊天窗口，帮我写出代码框架。”——这能帮你省下一半时间。
3.  **PandasAI 是捷径：**
    如果你想自己手写 Prompt 让 LLM 生成 Python 代码再 `exec()`，难度会大很多（Day 3 会卡死）。对于大二学生，直接调库 `PandasAI` 既能达到效果，又能解释清楚原理，是最高效的选择。
